# 自学推荐

执行 Learn 工作流，生成每日学习推荐报告。

## 执行步骤

### 1. 检查学习主题
读取 `wiki/recommendations/1-Day learn.md`：
- 检查"### 优先了解："章节下的内容
- **如果为空或无实质内容**：报告"今日无学习主题，任务结束"并退出
- **如果有内容**：继续执行后续步骤

### 2. 分析已有知识
搜索 wiki 目录，识别知识库中已有的相关内容：
- 使用 Grep 工具搜索关键词
- 读取相关摘要和概念页面
- 标记 ✅ 已有 / ❌ 缺失

### 3. 联网搜索
使用 tavily-search MCP 插件搜索技术博客：
- 搜索来源：CSDN/知乎/掘金/博客园/GitHub/官方文档
- 博客类别偏好：原理介绍、技术分析、架构图
- 每个主题搜索 5-10 篇博客

### 4. 生成推荐报告
保存至 `wiki/recommendations/YYYY-MM-DD.md`，格式：

```yaml
---
title: 学习推荐报告 - YYYY-MM-DD
created: YYYY-MM-DD
tags: [学习推荐, 每日报告]
---

# 学习推荐报告

## 一、用户指定主题
来自[[recommendations/1-Day learn]]

## 二、已有知识总结
标记 ✅ 已有 / ❌ 缺失

## 三、深入推荐
推荐博客表格

## 四、学习路径建议

## 五、知识图谱关联建议
```

### 5. 下载博客到 Self learn
使用 defuddle CLI 下载推荐的可访问博客：
- 保存路径：`raw/sources/Self learn/`
- 文件命名：`主题-来源-标题.md`
- 每个主题下载 3-5 篇最优质的博客
- 不可访问的链接跳过，另选替代

## 注意事项

- 只检查"优先了解"章节，"后续了解"不作为当日推荐依据
- 所有下载的博客标记为低可信度（Self learn 目录）
- 定时任务会在 23:00 自动执行 Ingest 处理新增博客
