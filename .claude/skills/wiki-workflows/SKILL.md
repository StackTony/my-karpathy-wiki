# Wiki 工作流技能

三个 Wiki 管理工作流的快捷命令。

## 可用技能

| 命令 | 触发词 | 功能 |
|------|--------|------|
| `/自建更新` | 自建更新 | 执行 Ingest 工作流，检查 raw/sources 变化并生成摘要 |
| `/自检索引` | 自检索引 | 执行 Lint 工作流，检查断链、孤立页面、索引一致性 |
| `/自学推荐` | 自学推荐 | 执行 Learn 工作流，读取学习主题并生成推荐报告 |

---

## 自建更新 (Ingest)

手动触发定时任务的检查更新功能。

**执行步骤**：
1. 检查 `raw/sources` 目录下是否有新增或修改的 `.md` 文件
2. 如果有变更，按照 Karpathy Wiki 的 Ingest 工作流处理：
   - 读取新增的源文件
   - 创建对应的 `wiki/summaries/` 摘要
   - 创建相关的 `wiki/concepts/` 概念页面
   - 更新 `wiki/index.md` 索引（注意去重）
   - 更新 `wiki/log.md` 日志
3. 使用 git status 或文件对比来判断变更
4. 如果没有变更，报告"本次检查无文档变更"

**可信度处理**：
- 高可信度来源（除 Self learn 外）：正常创建摘要
- 低可信度来源（Self learn）：摘要添加 `credibility: low` 字段

---

## 自检索引 (Lint)

执行 Wiki 健康检查。

**检查项**：
1. **断链问题**：wikilinks 指向不存在的页面
2. **孤立页面**：仅在 index.md 列出，缺少交叉引用的页面
3. **索引一致性**：统计数据与实际文件数量是否匹配
4. **知识矛盾**：页面之间是否存在矛盾陈述

**报告格式**：
- 问题列表（文件 + 断链/孤立原因）
- 建议修复方案
- 统计核实结果

## 自学推荐 (Learn)

执行每日学习推荐任务。

**执行步骤**：
1. **检查学习主题**：读取 `wiki/recommendations/1-Day learn.md`
   - 检查"### 优先了解："章节下的内容
   - 如果为空，报告"今日无学习主题，任务结束"并退出
2. **有指定主题时执行推荐**：
   - 分析知识库已有相关内容（搜索 wiki 目录）
   - 使用 tavily-search 联网搜索技术博客（偏好：原理介绍、技术分析、架构图）
   - 整理推荐报告，保存至 `wiki/recommendations/YYYY-MM-DD.md`
3. **下载博客到 Self learn 目录**：
   - 使用 defuddle CLI 下载推荐的可访问博客
   - 保存路径：`raw/sources/Self learn/`
   - 文件命名：`主题-来源-标题.md`
   - 每个主题下载 3-5 篇最优质的博客

**注意**：只检查"优先了解"章节，"后续了解"不作为当日推荐依据
