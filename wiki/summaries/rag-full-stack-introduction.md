# RAG 全栈介绍

RAG（Retrieval-Augmented Generation）是让大模型在生产环境真正用起来的核心技术。

---

## 一、为什么需要 RAG？

### 大模型的两大致命缺陷

| 缺陷 | 说明 | 影响 |
|------|------|------|
| **知识截止** | 训练数据有截止日期 | 不知道最新信息 |
| **幻觉** | 一本正经胡说八道 | 专业领域严重后果 |

### 传统方案的痛点

| 方案 | 问题 |
|------|------|
| 微调模型 | 费用几万到几百万，知识固化 |
| 重新训练 | 文档每天更新，无法频繁重训 |

---

## 二、RAG 核心思路

**一句话概括**：让大模型作答之前，先去外部知识库找相关信息，连同问题一起交给大模型。

### 三步流程

```
用户提问
    ↓
[Retrieval 检索] → 去知识库找相关文档片段
    ↓
[Augmentation 增强] → 把找到的内容拼到 Prompt 里
    ↓
[Generation 生成] → 大模型根据上下文生成答案
```

### 开卷考试类比

- 大模型 = 能看懂资料、组织语言作答的"学生"
- 外部知识库 = "参考书"

---

## 三、RAG 解决的问题

| 问题 | 状态 |
|------|------|
| 知识时效性 | ✅ 外部知识库随时更新，无需重训 |
| 幻觉抑制 | ✅ 答案有"依据"可查 |
| 私有知识接入 | ✅ 企业内部文档安全接入 |
| 可追溯性 | ✅ 答案附来源链接 |
| 成本可控 | ✅ 无需重训，只维护知识库 |

---

## 四、RAG 技术迭代

### 三代演进

| 代际 | 特点 | 技术特征 |
|------|------|----------|
| **Naive RAG** | 简单检索+生成 | 向量检索 → 拼接 → 生成 |
| **Advanced RAG** | 优化检索质量 | 混合检索、重排序、查询改写 |
| **Modular RAG** | 模块化架构 | 独立模块：检索/增强/生成/编排 |

### Advanced RAG 三层优化

| 层级 | 优化方向 | 具体技术 |
|------|----------|----------|
| **索引层** | 数据质量 | 切分策略、元数据索引、多粒度 |
| **检索层** | 检索精度 | 混合检索、重排序、查询扩展 |
| **生成层** | 生成质量 | Prompt优化、上下文压缩、引用标注 |

---

## 五、混合检索与重排序

### 检索方式对比

| 方式 | 原理 | 优势 | 劣势 |
|------|------|------|------|
| **稀疏检索 (BM25)** | 关键词匹配 | 精确匹配、快速 | 语义理解弱 |
| **稠密检索 (向量)** | 语义相似度 | 语义理解强 | 精确匹配弱 |
| **混合检索** | BM25 + 向量 | 兼顾精确与语义 | 计算开销大 |

### RRF 融合算法

[[concepts/rrf-algorithm]] - 倒数排名融合，无需归一化，仅凭排名合并多路结果。

---

## 六、RAG-Fusion 多查询扩展

将用户一个问题扩展为 5-10 个子问题：
- 消除单一查询偏差
- 覆盖不同表达方式
- 用 RRF 合并多路结果

## 七、生产级 RAG 架构

### 核心组件

```
┌─────────────────────────────────────────┐
│  用户提问                                │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  Query Processing                        │
│  - 查询改写                               │
│  - 查询扩展                               │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  Hybrid Retrieval                        │
│  - BM25 稀疏检索                          │
│  - 向量稠密检索                           │
│  - RRF 融合                               │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  Reranking                               │
│  - Cross-Encoder 精确重排                 │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  Context Processing                      │
│  - 上下文压缩                             │
│  - 元数据注入                             │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  LLM Generation                          │
│  - 答案生成                               │
│  - 来源标注                               │
└─────────────────────────────────────────┘
```

## 八、RAG 未来方向

| 方向 | 说明 |
|------|------|
| **Agentic RAG** | Agent 主动检索、迭代优化 |
| **GraphRAG** | 知识图谱增强，实体关系推理 |
| **Long-Context RAG** | 利用长上下文模型，减少切分损失 |
| **Self-RAG** | 模型自判断是否需要检索 |

---

## 九、实战落地路径

### 从零到一

1. **数据准备**：文档清洗、切分策略
2. **向量构建**：Embedding 模型选择、向量库搭建
3. **检索优化**：混合检索、重排序
4. **生成优化**：Prompt 设计、上下文管理
5. **评测体系**：检索指标、生成指标

### 评测指标

| 类型 | 指标 |
|------|------|
| 检索评估 | Recall@K、MRR、NDCG |
| 生成评估 | 事实性、完整性、可读性 |
