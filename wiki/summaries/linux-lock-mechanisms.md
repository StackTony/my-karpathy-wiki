## 概述

Linux内核提供了丰富的同步机制，从忙等待锁到无锁机制，覆盖各种并发场景。

## 锁机制总览

```
                    Linux 内核同步机制
                           │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
┌────▼────┐           ┌─────▼─────┐         ┌──────▼──────┐
│ 忙等待锁 │           │  睡眠锁    │         │  无锁机制    │
└────┬────┘           └─────┬─────┘         └──────┬──────┘
    │                      │                      │
┌────┴────┐           ┌─────┴─────┐          ┌─────┴─────┐
│spinlock │           │ mutex     │          │ RCU       │
│seqlock  │           │ semaphore │          │ atomic    │
│bit spin │           │ rwsem     │          │ per-cpu   │
└─────────┘           │ rt_mutex  │          │ seqcount  │
                      │ ww_mutex  │          │ lockless  │
                      └───────────┘          └───────────┘
```

## 核心对比表

| 锁类型 | 等待方式 | 上下文限制 | 开销 | 适用场景 |
|--------|----------|------------|------|----------|
| **Spinlock** | 忙等待 | 任意上下文 | 极小 | 中断、极短临界区 |
| **Mutex** | 睡眠 | 仅进程上下文 | 中等 | 一般互斥、可睡眠 |
| **RWLock** | 忙等待 | 任意上下文 | 小 | 读多写少、短临界区 |
| **RW Semaphore** | 睡眠 | 仅进程上下文 | 中等 | 读多写少、长临界区 |
| **Seqlock** | 无锁重试 | 任意上下文 | 小 | 读远多于写 |
| **RCU** | 无锁 | 任意上下文 | 读零 | 读极多写极少 |
| **Semaphore** | 睡眠 | 仅进程上下文 | 大 | 资源计数、同步 |
| **Atomic** | 原子操作 | 任意上下文 | 极小 | 简单计数/标志 |

## Spinlock（自旋锁）

### 核心原则

> **"快进快出，绝不睡眠"**

- 持锁时间极短（通常 < 100条指令，微秒级）
- 等待时间 < 上下文切换开销（~5-10μs）
- 可用于任意上下文（中断、软中断、进程）

### 实现演进

| 代际 | 实现 | 特点 |
|------|------|------|
| 第1代 | Test-And-Set | 无公平性，可能饥饿 |
| 第2代 | Ticket Lock | FIFO公平性，缓存竞争严重 |
| 第3代 | MCS/Queued Lock | 本地变量自旋，NUMA友好 |

### API选择决策树

```
是否与硬件中断共享数据？
  是 → spin_lock_irqsave
  否 → 是否与软中断共享数据？
        是 → spin_lock_bh
        否 → spin_lock
```

## Mutex（互斥锁）

### 核心特性

- 睡眠锁，竞争失败时进程睡眠
- 仅限进程上下文使用
- 支持乐观自旋优化
- 支持优先级继承（避免优先级反转）

### Owner字段编码

```
┌─────────────────────────────────────────────┬─────┐
│         task_struct 指针 (高位)              │flags│
│                                             │ 低3位│
└─────────────────────────────────────────────┴─────┘
                              │
         0x01: MUTEX_FLAG_WAITERS (有等待者)
         0x02: MUTEX_FLAG_HANDOFF  (移交锁)
```

## RCU（Read-Copy-Update）

### 核心思想

> **"读者零开销"**

```
传统锁: lock → 读 → unlock（即使无冲突也有开销）
RCU:    直接读（无锁！）→ 读侧几乎零开销
代价:   写者延迟释放数据 → 内存短暂增加
```

### 工作流程

```
写者:
  复制 → 修改副本 → 替换指针 → 等待宽限期 → 释放旧数据

读者:
  rcu_read_lock() → rcu_dereference() → 使用数据 → rcu_read_unlock()
```

### RCU变体

| 变体 | 读侧特点 | 适用场景 |
|------|----------|----------|
| Classic RCU | 不可睡眠 | 内核通用同步 |
| SRCU | 可睡眠 | 需要在读侧睡眠 |
| RCU-sched | 禁用抢占 | 调度器相关 |
| RCU-bh | 禁用软中断 | 网络软中断路径 |

## 选择口诀

```
中断上下文用自旋，进程上下文看时间。
极短临界自旋锁，长久睡眠选 mutex。
读多写少分两路，短用 rwlock 长用 rwsem。
读极多时上 RCU，时间统计用 seqlock。
资源计数用信号量，简单计数原子化。
性能极致 per-cpu，无锁编程是王道。
```
