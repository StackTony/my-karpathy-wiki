## 定义

崩溃分析中通过寄存器值定位问题根因的技术。

## x86_64 寄存器

### 通用寄存器
| 寄存器 | 用途 |
|--------|------|
| RAX | 函数返回值、算术运算 |
| RDI | **第1个参数** |
| RSI | **第2个参数** |
| RDX | **第3个参数** |
| RCX | **第4个参数** |
| RBP | **栈帧基址**（定位局部变量） |
| RSP | **栈指针**（栈顶） |
| RIP | **下一条指令地址**（崩溃关键） |

### 函数调用约定
| 参数位置 | 寄存器 |
|----------|--------|
| 第1-6参数 | RDI, RSI, RDX, RCX, R8, R9 |
| 返回值 | RAX |
| 更多参数 | 栈传递 |

### 段寄存器
- GS: 内核存储每 CPU 变量

## ARM64 寄存器

### 通用寄存器
| 寄存器 | 用途 |
|--------|------|
| X0-X7 | **参数寄存器**，X0 也是返回值 |
| X29/FP | 栈帧基址（= x86 RBP） |
| X30/LR | **返回地址**（函数调用后返回） |
| SP | 栈指针（= x86 RSP） |
| PC | 程序计数器（= x86 RIP） |

### 函数调用约定
| 参数位置 | 寄存器 |
|----------|--------|
| 第1-8参数 | X0-X7 |
| 返回值 | X0 |
| 更多参数 | 栈传递 |

### 异常级别
| EL | 用途 |
|----|------|
| EL0 | 用户态 |
| EL1 | **内核态** |
| EL2 | Hypervisor |
| EL3 | Secure Monitor |

## 架构对比速查

| 功能 | x86_64 | ARM64 |
|------|--------|-------|
| 程序计数器 | RIP | PC |
| 栈指针 | RSP | SP |
| 帧指针 | RBP | X29 (FP) |
| 返回地址 | 栈上保存 | X30 (LR) |
| 第1-6参数 | RDI, RSI, RDX, RCX, R8, R9 | X0-X5 |
| 返回值 | RAX | X0 |
| 内核态 | Ring 0 | EL1 |

## 常见错误模式

### 空指针解引用
```
BUG: kernel NULL pointer dereference at 0x123
```
- 地址小数值（如 0x123）→ 结构体成员偏移处指针为 NULL

### 非法地址访问
```
BUG: kernel paging request at ffffdeadbeef0000
```
- 毒值地址（如 0xdeadbeef）→ UAF（使用后释放）

### 栈溢出
```
stack segment: 0000 [#1] SMP
```
- SP 指向非法地址 → 无限递归或局部变量过大

## ARM64 栈回溯

栈帧结构：
```
+------------------+ <- 高地址
|  局部变量        |
+------------------+
|  保存的 FP (X29) | <- 当前 FP 指向这里
+------------------+
|  保存的 LR (X30) |
+------------------+ <- 低地址 (SP)
```

回溯过程：`FP → 保存的FP → 保存的LR → 上一级FP → ...`
